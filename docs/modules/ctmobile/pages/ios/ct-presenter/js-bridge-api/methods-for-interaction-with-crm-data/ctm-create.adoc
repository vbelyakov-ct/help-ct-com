= ctm.create

The method of xref:ios/ct-presenter/js-bridge-api/index.adoc[JS Bridge] functionality is intended to bulk create records of offline objects:

* xref:ios/getting-started/application-permission-settings.adoc[the user profile] should have permission to create records of the required object;
* xref:ios/admin-guide/managing-offline-objects/index.adoc#h2_1747811907[permission to create] the required offline object is allowed.

[[h2_905713055]]
== Query Format

[source,javascript]
----
ctm.create([SObject], handler)
----

[.apiobject]#SObject# is the JS object that indicates the API name of the offline object and a list of fields with values.

[[h2_442663712]]
== Handler

include::partial$ctm-methods-handler.adoc[]

[source,javascript]
----
ctm.create([{"SObject": "Account", "Name": "Google, Inc"}])
----

[source,javascript]
----
var sObjArray = [];
var sObj = {
SObject: "Account",
Name: "Test Account",
Phone: "123-45-67",
Type: "Prospect"
};
sObjArray.push(sObj);

ctm.create(sObjArray, function(createResponse){
console.log('New record Id', createResponse.result[0].Id);
})
----

[source,javascript]
----
var createString = '[{"SObject":"Account", "Name":"Test Account"}, {"SObject":"Account", "Name":"Sample Account"}]';

var createObject = JSON.parse(createString);

ctm.create(createObject, function(response){
console.log('Create result', response);
})
----

The result:

----
{
    "result": [
        {
            "Id": "DSFS-2342-TYRE-ADSFBA374Q8RQ8",
            "Name": "Google, Inc"
        }
    ],
    "success": true,
}
